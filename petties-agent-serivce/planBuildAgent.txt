Agent Query: User-facing conversational interface. Conversational interface for interacting with the Main Agent about any system information: patient (pets of user), medical record of pet, data retrieval, detailed patient analysis, etc. Users send requests directly to the Main Agent (Supervisor), which reasons through the task and creates an execution plan. All tasks run asynchronously in the background.

Develop a comprehensive **Admin Dashboard Page** within Agent Studio that:

1. **Provides a dedicated UI** for:
   - Testing and debugging individual agents and tools
   - Manually triggering agent workflows
   - Visualizing results, logs, and reasoning steps
   - Simulating user queries for chatbot interactions

2. **Enables seamless chatbot integration** for users by:
   - Connecting the backend agent orchestration to live conversational UI
   - Allowing dynamic simulated conversations to test response accuracy
   - Supporting API endpoints for real-time message exchange

3. **Incorporates MCP (Module Control Platform) server** as:
   - A core orchestrator and plugin system
   - A customizable toolkit to self-build and extend MCP modules/tools
   - A backend module loader, updater, and sandbox environment
   - Accessible via API endpoints that can be wrapped as tools

4. **Design considerations**:
   - Frontend built with React and Ant Design or Material UI for professional look
   - Backend API support in FastAPI with endpoints for:
     - Triggering agent workflows
     - Testing individual agents/tools
     - Managing MCP modules (list, deploy, update)
     - Chatbot message flow simulation
   - MCP server API to facilitate:
     - Self-configuration (add/remove/update modules)
     - Plugin execution sandbox
     - Monitoring resource usage & logs

5. **Workflow flow**:
   - Admin page tests agents and tools live, displaying reasoning process
   - Admin can register or uninstall MCP modules dynamically
   - MCP modules can be built and integrated as tools, invoked through API
   - Chatbot interactions leverage agent and MCP backend, live streaming responses

6. **Styling & UX**:
   - Use a clean, professional design emphasizing usability
   - Clear buttons for "Run Test", "Simulate Chat", "Manage MCP Modules"
   - Show logs, reasoning chains, and results in collapsible panels
   - Responsive layout for desktop and tablet

This setup ensures the system is **fully testable**, **extensible**, and **ready for user-facing chatbot deployment**, fully leveraging MCP server's flexible plugin architecture for easy module management and custom tool building.
